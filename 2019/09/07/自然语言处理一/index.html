常用计算复杂度的两种方法

主定理公式

1.主定理的使用范围

对于规模为n的分治问题(递归)即时间可以表示为T(n)=aT(n/b)+f(n)的形式

2.判定方法

1.O(nlog_b_a) > O(f(n))===============>T(n)=O(nlog_b_a)
2.O(nlog_b_a) < O(f(n))===============>T(n)=O(f(n))
3.O(nlog_b_a*（log_n)^k)=O(f(n))======>T(n)=O(nlog_b_a*(logn)^(k+1))

3.案例分析

例如对于A=[3,4,1,6,7,2,5,9]采用归并排序,假设时间T(n)
1.[3,4,1,6],[7,2,5,9]  算法所用的时间为T(n)=2T(n/2)+n
    其中n:表示对于两个数组，一共要比较n次才能够获得排序后的队列
计算过程
    b=2,a=2,f(n)=n    
    n^log_b_a=n=f(n)
    所以T(n)=O(nlogn)

递归数的方式

1.适用范围:

对于递归嵌套调用，都可以画出递归树结构，然后计算复杂度

2.案例分析

计算斐波那契数列的时间复杂度
f(n)=f(n-1) + f(n-2)
f(n-1)=f(n-2) + f(n-3)....
画出递归数，我们可以看到
第0层    2^0
第一层    2^1
第二层    2^2
...
第n-2层    2^(n-2)
总时间复杂度2^0+2^1+2^2+....+2^(n-2)，近似为O(2^n)

斐波那契数列的改进方案：
    对于上面的计算，有很多部分重复，我们可以采用DP方法O(n)

P(可以解决的问题时间复杂度O(n^p))
NP(可以解决的问题，但是用于判断这个解是不是想要的这类问题)
